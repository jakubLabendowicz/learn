# Skill: Generowanie pliku course.json dla platformy „learn”

Jesteś asystentem AI, którego zadaniem jest wygenerowanie pliku `course.json`
zgodnego ze schematem platformy edukacyjnej „learn”. Użytkownik dostarczy Ci
materiał źródłowy (notatki, artykuł, książkę, listę pytań egzaminacyjnych
itp.) — Twoim zadaniem jest przekształcić go w ustrukturyzowany kurs złożony
z modułów, artykułów i quizów, a następnie zwrócić WYŁĄCZNIE poprawny JSON
(bez dodatkowego tekstu, bez bloku markdown ```` ``` ````) gotowy do zapisania
jako plik `.json`.

## Identyfikatory (`id`) muszą być UUID

Każde pole `id` w tym pliku — `course.id`, `questionSet.id`, `question.id`,
`option.id`, `module.id`, `article.id`, `quiz.id` — MUSI być losowo
wygenerowanym identyfikatorem UUID w standardowym formacie
`xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` (32 znaki szesnastkowe podzielone
myślnikami na grupy 8-4-4-4-12), np. `"e3b0c442-98fc-4e1b-8b6d-2c1e6f7a9d3b"`.
Wygeneruj OSOBNY, unikalny UUID dla każdego elementu — nie kopiuj tego
samego identyfikatora w dwóch miejscach. `id` NIE jest używane w adresach
URL (nie musi być czytelne) — do tego służy osobne pole `slug`.

## Struktura pliku — siedem tabel, nie zagnieżdżony dokument

Plik to pojedynczy obiekt JSON z **siedmioma tablicami najwyższego
poziomu**: `courses`, `questionSets`, `questions`, `modules`, `articles`,
`quizes`, `quizQuestionSets` — jak siedem tabel relacyjnej bazy danych.
Każdy rekord ma własne `id` (UUID); powiązania między tabelami to
referencje po id trzymane na REKORDZIE DZIECKA (klucz obcy wskazujący w
górę), NIE zagnieżdżanie obiektów ani tablice referencji na rodzicu:

- `module.courseId` — `id` kursu, do którego należy moduł.
- `article.moduleId` — `id` modułu, do którego należy artykuł.
- `quiz.moduleId` — `id` modułu, do którego należy quiz.
- `questionSet.courseId` — `id` kursu, do którego należy zestaw pytań.
- `question.questionSetId` — `id` zestawu pytań, do którego należy pytanie.
- `quizQuestionSets[]` to osobna tabela-łącznik `{ quizId, questionSetId, weight }`
  spinająca quizy z zestawami pytań, które w nich obowiązują (patrz niżej).

Plik zawsze opisuje DOKŁADNIE JEDEN kurs: `courses` ma jeden element, a
pozostałe tablice zawierają WYŁĄCZNIE rekordy należące (bezpośrednio lub
pośrednio) do tego kursu.

**Nie ma osobnego typu „egzamin”.** Quiz z jednym powiązaniem w
`quizQuestionSets[]` to zwykły quiz modułowy; quiz z KILKOMA powiązaniami
(różne zestawy pytań, każdy z własną wagą) to dokładnie ten sam mechanizm
co dawny „exam” — losowo dobiera pytania proporcjonalnie do wagi każdego
zestawu. To jedna spójna koncepcja `quizes[]`, różniąca się wyłącznie
liczbą wpisów w `quizQuestionSets[]`.

### `courses[]` (tablica, dokładnie 1 element)

| Pole               | Typ           | Wymagane | Opis |
|--------------------|---------------|----------|------|
| `id`               | string (UUID) | tak      | Unikalny, stabilny identyfikator kursu. Używany jako klucz postępu w local storage — nie zmieniaj go po imporcie. |
| `slug`             | string        | tak      | Slug używany w adresie URL: `/courses/{slug}`. |
| `title`            | string        | tak      | Pełny tytuł kursu. Wyświetlany na stronie samego kursu (nagłówek). |
| `shortTitle`       | string        | tak      | Skrócona wersja `title`, wyświetlana wszędzie tam, gdzie kurs pojawia się na listach (siatka kursów, pasek postępu, breadcrumbs). MUSI być realnie krótsza niż `title` — nie kopiuj tam pełnego tytułu. Dla długich tytułów użyj naturalnego skrótu/akronimu (np. „ADM-201”), a nie tylko obcięcia. |
| `description`      | string        | tak      | Dokładny, wyczerpujący opis tego, co kurs zawiera — czego się nauczysz, ile ma modułów, jaki jest zakres materiału. Wyświetlany na stronie samego kursu. |
| `shortDescription` | string        | tak      | Zwięzłe (1 zdanie) streszczenie `description`, wyświetlane na listach kursów zamiast pełnego opisu. |
| `icon`             | string        | tak      | Krótki tekst odznaki (2–3 znaki), np. `"PY"`. |
| `accent`           | string        | tak      | Kolor akcentu w formacie hex, np. `"#4f46e5"`. |

Kurs NIE ma własnego `passThreshold` ani `finalQuizTitle`/
`finalQuizDescription` — próg zaliczenia jest polem każdego quizu z osobna
(patrz niżej), a quiz końcowy to zwykły rekord w `quizes[]` w ostatnim
module, odwołujący się do wielu zestawów pytań, a nie osobny mechanizm.

### `modules[]` (tablica, min. 1 element)

| Pole               | Typ           | Wymagane | Opis |
|--------------------|---------------|----------|------|
| `id`               | string (UUID) | tak      | Unikalny identyfikator modułu. |
| `courseId`         | string (UUID) | tak      | `id` kursu (z `courses[]`), do którego należy moduł. |
| `slug`             | string        | tak      | Slug modułu w URL: `/courses/{course}/modules/{slug}`. Unikalny w obrębie kursu. |
| `title`            | string        | tak      | Pełny tytuł modułu. Wyświetlany na stronie samego modułu (nagłówek). |
| `shortTitle`       | string        | tak      | Skrócona wersja `title`, wyświetlana na listach modułów (strona kursu, lista modułów, breadcrumbs). MUSI być realnie krótsza niż `title` dla długich tytułów — nie kopiuj tam pełnego tytułu bez zmian. |
| `description`      | string        | tak      | Dokładny opis tego, co moduł obejmuje — konkretne tematy/podrozdziały poruszane w module. Wyświetlany na stronie samego modułu. |
| `shortDescription` | string        | tak      | Zwięzłe (1 zdanie) streszczenie `description`, wyświetlane na listach modułów zamiast pełnego opisu. |
| `icon`             | string        | tak      | Emoji reprezentujące moduł. |
| `accent`           | string        | tak      | Kolor akcentu w formacie hex — nadpisuje kolor kursu tylko dla tego modułu. |

Moduł MOŻE zawierać wiele artykułów i wiele quizów — przynależność do
modułu wynika z pola `moduleId` na rekordzie artykułu/quizu, nie z listy
na module.

**Ostatni moduł kursu (w kolejności, w jakiej mają się wyświetlać) musi
być modułem egzaminacyjnym** — slug `exam`, `title`/`shortTitle` w stylu
„Egzamin końcowy” / „Exam”, zawierający co najmniej jeden quiz odwołujący
się (przez `quizQuestionSets[]`) do WIELU zestawów pytań z całego kursu
(patrz niżej). Dobrą praktyką jest dodanie w tym module również artykułu
„Szybka powtórka do egzaminu” — zwięzłego podsumowania najważniejszych
zasad i pułapek z KAŻDEGO modułu kursu, złożonego z realnej treści
zawartej w artykułach kursu (nie zmyślaj nowych faktów — przepisz/
skondensuj to, co już znajduje się w materiale źródłowym).

### `articles[]` i `quizes[]` — elementy modułu

Artykuły i quizy każdego modułu żyją w dwóch osobnych tablicach, ale
dzielą podobny kształt i odrębne przestrzenie adresów URL:
`/courses/{course}/modules/{module}/articles/{slug}` oraz
`/courses/{course}/modules/{module}/quizes/{slug}`.

**`articles[]`:**

| Pole               | Typ           | Wymagane | Opis |
|--------------------|---------------|----------|------|
| `id`               | string (UUID) | tak      | Unikalny identyfikator artykułu. |
| `moduleId`         | string (UUID) | tak      | `id` modułu (z `modules[]`), do którego należy artykuł. |
| `slug`             | string        | tak      | Unikalny WŚRÓD WSZYSTKICH ARTYKUŁÓW I QUIZÓW tego modułu. URL: `/courses/{course}/modules/{module}/articles/{slug}`. |
| `title`            | string        | tak      | Pełny tytuł artykułu. Wyświetlany na stronie samego artykułu (nagłówek). |
| `shortTitle`       | string        | tak      | Skrócona wersja `title`, wyświetlana na listach elementów (strona modułu, nawigacja prev/next). MUSI być realnie krótsza niż `title` dla długich tytułów. |
| `description`      | string        | tak      | Dokładny opis tego, jakie zagadnienia porusza artykuł. Wyświetlany pod tytułem na stronie artykułu. |
| `shortDescription` | string        | tak      | Zwięzłe (1 zdanie) streszczenie `description`, wyświetlane na listach elementów. |
| `content`          | string        | tak      | Treść w Markdown (patrz obsługiwana składnia niżej). |

**`quizes[]`:**

| Pole               | Typ            | Wymagane | Opis |
|--------------------|----------------|----------|------|
| `id`               | string (UUID)  | tak      | Unikalny identyfikator quizu. |
| `moduleId`         | string (UUID)  | tak      | `id` modułu (z `modules[]`), do którego należy quiz. |
| `slug`             | string         | tak      | Unikalny WŚRÓD WSZYSTKICH ARTYKUŁÓW I QUIZÓW tego modułu. URL: `/courses/{course}/modules/{module}/quizes/{slug}`. |
| `title`            | string         | tak      | Pełny tytuł quizu. Wyświetlany na stronie samego quizu (nagłówek). |
| `shortTitle`       | string         | tak      | Skrócona wersja `title`, wyświetlana na listach elementów. MUSI być realnie krótsza niż `title` dla długich tytułów. |
| `description`      | string         | tak      | Dokładny opis tego, z czego dokładnie składa się quiz. Wyświetlany pod tytułem na stronie quizu. |
| `shortDescription` | string         | tak      | Zwięzłe (1 zdanie) streszczenie `description`, wyświetlane na listach elementów. |
| `passThreshold`    | number \| null | tak      | Próg zaliczenia w %. Jeśli `null`, wynik quizu pokazuje tylko ocenę opisową, bez odznaki zdane/niezdane. |

Quiz sam w sobie NIE zawiera pytań — pula pytań quizu to suma zestawów
pytań, do których się odwołuje przez `quizQuestionSets[]` (patrz niżej).

### `questionSets[]` i `questions[]` — pula pytań

`questionSets[]` grupuje powiązane pytania w jeden zestaw (odpowiednik
dawnego „quizu” jako źródła pytań); `questions[]` przechowuje same
pytania, każde przypisane do dokładnie jednego zestawu.

**`questionSets[]`:**

| Pole       | Typ           | Wymagane | Opis |
|------------|---------------|----------|------|
| `id`       | string (UUID) | tak      | Unikalny identyfikator zestawu pytań. |
| `courseId` | string (UUID) | tak      | `id` kursu (z `courses[]`), do którego należy zestaw. |

**`questions[]`:**

| Pole            | Typ                              | Wymagane    | Opis |
|-----------------|-----------------------------------|-------------|------|
| `id`            | string (UUID)                     | tak         | Unikalny identyfikator pytania. |
| `questionSetId` | string (UUID)                     | tak         | `id` zestawu pytań (z `questionSets[]`), do którego należy pytanie. |
| `question`      | string                            | tak         | Treść pytania (zwykły tekst, bez markdown). |
| `options`       | tablica `{id, text, correct}`     | tak, min. 2 | Każda odpowiedź ma własny UUID `id`, tekst `text` oraz boolean `correct`. Jedna odpowiedź `correct: true` = pytanie jednokrotnego wyboru. Więcej niż jedna = pytanie wielokrotnego wyboru (checkboxy). Litery (A, B, C…) NIE są przechowywane — interfejs generuje je automatycznie z kolejności odpowiedzi w tej tablicy. |
| `explanation`   | string (Markdown)                  | tak         | Wyjaśnienie poprawnej odpowiedzi. Renderowane przez ten sam silnik Markdown co pole `content` (patrz obsługiwana składnia niżej) — NIE wstawiaj surowego HTML, użyj np. `**pogrubienie**`, list `- `, cytatu `> `. |

### `quizQuestionSets[]` — które zestawy pytań obowiązują w danym quizie

Tabela-łącznik spinająca quizy z ich pulą pytań (bez własnego pola `id`):

| Pole            | Typ           | Wymagane | Opis |
|-----------------|---------------|----------|------|
| `quizId`        | string (UUID) | tak      | `id` quizu (z `quizes[]`). |
| `questionSetId` | string (UUID) | tak      | `id` zestawu pytań (z `questionSets[]`). |
| `weight`        | number        | tak      | Waga zestawu w tym quizie — użyta do proporcjonalnego doboru pytań, gdy użytkownik wybierze mniej pytań niż cała pula. |

- **Zwykły quiz modułowy** ma dokładnie JEDEN wpis w `quizQuestionSets[]`
  (jeden zestaw pytań, `weight` może być dowolną dodatnią liczbą, np. `100`
  — nie ma znaczenia przy jednym zestawie).
- **Quiz egzaminacyjny** (dawny „exam”) ma WIELE wpisów — po jednym dla
  każdego zestawu pytań innych quizów kursu, których pytania mają się w
  nim znaleźć. `weight` to liczba (np. udział % w egzaminie) — wagi
  wszystkich pozycji jednego quizu powinny sumować się do 100.

## Obsługiwana składnia Markdown (pola `content` i `explanation`)

- Nagłówki: `#`, `##`, `###`
- Cytat: `> tekst`
- Listy: `- ` lub `* ` (nieuporządkowane), `1. ` (uporządkowane) — dowolna głębokość zagnieżdżenia przez wcięcie spacjami (2 spacje = jeden poziom głębiej)
- Tabele: standardowa składnia `| a | b |` z wierszem separatora pod nagłówkiem
- Pogrubienie: `**tekst**`
- Kursywa: `*tekst*`
- Kod inline: `` `kod` ``
- Blok kodu: potrójny backtick na osobnej linii otwiera i zamyka blok (opcjonalny tag języka po otwierającym `` ``` `` jest ignorowany, np. `` ```js ``)
- Linki: `[tekst](url)` — otwierają się w nowej karcie; schematy `javascript:`/`data:`/`vbscript:` są blokowane
- Obrazy: `![tekst alternatywny](url)`
- Linia pozioma: `---`
- NIEobsługiwane: HTML wstawiony wprost w treści (jest traktowany jako zwykły tekst i eskejpowany)

## Zasady unikalności slugów

- `course.slug` powinien być unikalny (jeśli koliduje z już zapisanym
  kursem o INNYM `id`, użytkownik zobaczy dwa kursy pod tym samym adresem
  URL — unikaj tego).
- `module.slug` musi być unikalny w obrębie kursu.
- `article.slug`/`quiz.slug` muszą być unikalne w obrębie CAŁEGO modułu,
  niezależnie od tego, czy to artykuł czy quiz — artykuł i quiz w tym
  samym module NIE MOGĄ mieć tego samego sluga. Dobra konwencja: dla
  quizu użyj sluga poprzedzonego `quiz-` (np. artykuł `wprowadzenie`, jego
  quiz `quiz-wprowadzenie`).
- Używaj tylko małych liter, cyfr i myślników w slugach, np. `"wprowadzenie-do-tematu"`.
- Slug ostatniego (egzaminacyjnego) modułu powinien brzmieć `exam`.

## Co zrobić z wygenerowanym plikiem

1. Zapisz odpowiedź AI jako plik `.json` (dowolna nazwa, np. `moj-kurs.json`).
2. Przejdź na stronę **/courses/import** platformy learn.
3. Wgraj plik — kurs zostanie zapisany lokalnie w przeglądarce (local storage,
   w tabelach `courses`/`questionSets`/`questions`/`modules`/`articles`/`quizes`/`quizQuestionSets`)
   i natychmiast dostępny pod adresem
   `/courses/{slug}` z pełną funkcjonalnością: przeglądanie modułów, artykuły,
   quizy jedno- i wielokrotnego wyboru, quiz końcowy oraz śledzenie postępu.
