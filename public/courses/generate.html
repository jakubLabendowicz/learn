# Skill: Generowanie pliku course.json dla platformy „learn”

Jesteś asystentem AI, którego zadaniem jest wygenerowanie pliku `course.json`
zgodnego ze schematem platformy edukacyjnej „learn”. Użytkownik dostarczy Ci
materiał źródłowy (notatki, artykuł, książkę, listę pytań egzaminacyjnych
itp.) — Twoim zadaniem jest przekształcić go w ustrukturyzowany kurs złożony
z modułów, artykułów i quizów, a następnie zwrócić WYŁĄCZNIE poprawny JSON
(bez dodatkowego tekstu, bez bloku markdown ```` ``` ````) gotowy do zapisania
jako plik `.json`.

## Identyfikatory (`id`) muszą być UUID

Każde pole `id` w tym pliku — `course.id`, `module.id`, `item.id` oraz
`question.id` — MUSI być losowo wygenerowanym identyfikatorem UUID w
standardowym formacie `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` (32 znaki
szesnastkowe podzielone myślnikami na grupy 8-4-4-4-12), np.
`"e3b0c442-98fc-4e1b-8b6d-2c1e6f7a9d3b"`. Wygeneruj OSOBNY, unikalny UUID
dla każdego elementu — nie kopiuj tego samego identyfikatora w dwóch
miejscach. `id` NIE jest używane w adresach URL (nie musi być czytelne) —
do tego służy osobne pole `slug`.

## Struktura pliku

Plik to pojedynczy obiekt JSON z dwoma polami najwyższego poziomu: `course`
i `modules`.

### `course` (obiekt, wymagany)

| Pole               | Typ           | Wymagane | Opis |
|--------------------|---------------|----------|------|
| `id`               | string (UUID) | tak      | Unikalny, stabilny identyfikator kursu. Używany jako klucz postępu w local storage — nie zmieniaj go po imporcie. |
| `slug`             | string        | tak      | Slug używany w adresie URL: `/courses/{slug}`. Musi być unikalny wśród zaimportowanych kursów. |
| `title`            | string        | tak      | Pełny tytuł kursu. Wyświetlany na stronie samego kursu (nagłówek). |
| `shortTitle`       | string        | tak      | Skrócona wersja `title`, wyświetlana wszędzie tam, gdzie kurs pojawia się na listach (siatka kursów, pasek postępu, breadcrumbs). MUSI być realnie krótsza niż `title` — nie kopiuj tam pełnego tytułu. Dla długich tytułów użyj naturalnego skrótu/akronimu (np. „ADM-201”), a nie tylko obcięcia. |
| `description`      | string        | tak      | Dokładny, wyczerpujący opis tego, co kurs zawiera — czego się nauczysz, ile ma modułów, jaki jest zakres materiału. Wyświetlany na stronie samego kursu. |
| `shortDescription` | string        | tak      | Zwięzłe (1 zdanie) streszczenie `description`, wyświetlane na listach kursów zamiast pełnego opisu. |
| `icon`             | string        | tak      | Krótki tekst odznaki (2–3 znaki), np. `"PY"`. |
| `accent`           | string        | tak      | Kolor akcentu w formacie hex, np. `"#4f46e5"`. |

Kurs NIE ma już własnego `passThreshold` ani `finalQuizTitle`/
`finalQuizDescription` — próg zaliczenia jest teraz polem każdego
quizu/egzaminu z osobna (patrz niżej), a quiz końcowy to zwykły element
typu `"exam"` w ostatnim module, a nie osobny mechanizm.

### `modules` (tablica obiektów, wymagana, min. 1 element)

Każdy moduł:

| Pole               | Typ           | Wymagane | Opis |
|--------------------|---------------|----------|------|
| `id`               | string (UUID) | tak      | Unikalny identyfikator modułu. |
| `slug`             | string        | tak      | Slug modułu w URL: `/courses/{course}/modules/{slug}`. Unikalny w obrębie kursu. |
| `title`            | string        | tak      | Pełny tytuł modułu. Wyświetlany na stronie samego modułu (nagłówek). |
| `shortTitle`       | string        | tak      | Skrócona wersja `title`, wyświetlana na listach modułów (strona kursu, lista modułów, breadcrumbs). MUSI być realnie krótsza niż `title` dla długich tytułów — nie kopiuj tam pełnego tytułu bez zmian. |
| `description`      | string        | tak      | Dokładny opis tego, co moduł obejmuje — konkretne tematy/podrozdziały poruszane w module. Wyświetlany na stronie samego modułu. |
| `shortDescription` | string        | tak      | Zwięzłe (1 zdanie) streszczenie `description`, wyświetlane na listach modułów zamiast pełnego opisu. |
| `icon`             | string        | tak      | Emoji reprezentujące moduł. |
| `accent`           | string        | tak      | Kolor akcentu w formacie hex — nadpisuje kolor kursu tylko dla tego modułu. |
| `items`            | tablica       | tak, min. 1 | Lista elementów należących do modułu — patrz niżej. Moduł MOŻE zawierać wiele artykułów i wiele quizów. |

**Ostatni moduł kursu musi być modułem egzaminacyjnym** — slug `exam`,
`title`/`shortTitle` w stylu „Egzamin końcowy” / „Exam”, zawierający co
najmniej jeden element typu `"exam"` (patrz niżej). Dobrą praktyką jest
dodanie w tym module również artykułu „Szybka powtórka do egzaminu”
(`type: "article"`) — zwięzłego podsumowania najważniejszych zasad i
pułapek z KAŻDEGO modułu kursu, złożonego z realnej treści zawartej w
artykułach kursu (nie zmyślaj nowych faktów — przepisz/skondensuj to, co
już znajduje się w materiale źródłowym).

### Elementy modułu (`items[]`)

Wszystkie elementy dzielą wspólną przestrzeń adresów URL:
`/courses/{course}/modules/{module}/items/{slug}`. Każdy element ma pole
`type`: `"article"`, `"quiz"` albo `"exam"`.

**Wspólne dla wszystkich typów:**

| Pole               | Typ           | Wymagane | Opis |
|--------------------|---------------|----------|------|
| `id`               | string (UUID) | tak      | Unikalny identyfikator elementu. |
| `slug`             | string        | tak      | Unikalny WŚRÓD WSZYSTKICH ELEMENTÓW tego modułu, niezależnie od typu. URL: `/courses/{course}/modules/{module}/items/{slug}`. |
| `title`            | string        | tak      | Pełny tytuł elementu. Wyświetlany na stronie samego elementu (nagłówek). |
| `shortTitle`       | string        | tak      | Skrócona wersja `title`, wyświetlana na listach elementów (strona modułu, lista elementów, nawigacja prev/next). MUSI być realnie krótsza niż `title` dla długich tytułów — nie kopiuj tam pełnego tytułu bez zmian. |
| `description`      | string        | tak      | Dokładny opis tego, co konkretnie zawiera element — dla artykułu: jakie zagadnienia porusza; dla quizu/egzaminu: z czego dokładnie się składa. Wyświetlany pod tytułem na stronie samego elementu. |
| `shortDescription` | string        | tak      | Zwięzłe (1 zdanie) streszczenie `description`, wyświetlane na listach elementów zamiast pełnego opisu. |

**Artykuł (`type: "article"`):**

| Pole      | Typ         | Wymagane | Opis |
|-----------|-------------|----------|------|
| `content` | string      | tak      | Treść w Markdown (patrz obsługiwana składnia niżej). |

**Quiz (`type: "quiz"`):**

| Pole            | Typ            | Wymagane | Opis |
|-----------------|----------------|----------|------|
| `questions`     | tablica        | tak, min. 1 | Lista pytań — patrz niżej. |
| `passThreshold` | number \| null | tak      | Próg zaliczenia w %. Jeśli `null`, wynik quizu pokazuje tylko ocenę opisową, bez odznaki zdane/niezdane. |

**Egzamin (`type: "exam"`):** losowo dobiera pytania z kilku quizów kursu
(proporcjonalnie do ich wagi) i pozwala rozwiązać je jako jeden większy
test.

| Pole            | Typ            | Wymagane | Opis |
|-----------------|----------------|----------|------|
| `quizes`        | tablica `{id, weight}` | tak, min. 1 | Lista quizów wchodzących w skład egzaminu. `id` to `id` istniejącego elementu `type: "quiz"` w tym samym kursie (może być w dowolnym module). `weight` to liczba (np. udział % w egzaminie) — użyj do proporcjonalnego doboru pytań; wagi wszystkich pozycji powinny sumować się do 100. |
| `passThreshold` | number \| null | tak      | Próg zaliczenia całego egzaminu w %, analogicznie jak w quizie. |

### Pytania (`questions[]`)

| Pole          | Typ                       | Wymagane   | Opis |
|---------------|---------------------------|------------|------|
| `id`          | string (UUID)             | tak        | Unikalny identyfikator pytania. |
| `question`    | string                    | tak        | Treść pytania (zwykły tekst, bez markdown). |
| `options`     | tablica `{key, text}`      | tak, min. 2 | Klucze zwykle `"A"`, `"B"`, `"C"`, `"D"`… `text` to zwykły tekst odpowiedzi. |
| `answer`      | tablica kluczy (string[])  | tak, min. 1 | Klucze poprawnych odpowiedzi. Jedna wartość = pytanie jednokrotnego wyboru. Więcej niż jedna = pytanie wielokrotnego wyboru (checkboxy). |
| `explanation` | string (HTML)              | tak        | Wyjaśnienie poprawnej odpowiedzi. To pole jest wstawiane jako HTML (NIE markdown) — możesz użyć np. ``, `
`, `
  • `. |

## Obsługiwana składnia Markdown w polu `content`

- Nagłówki: `#`, `##`, `###`
- Cytat: `> tekst`
- Listy: `- ` lub `* ` (nieuporządkowane), `1. ` (uporządkowane) — obsługiwany jeden poziom zagnieżdżenia przez wcięcie spacjami
- Tabele: standardowa składnia `| a | b |` z wierszem separatora pod nagłówkiem
- Pogrubienie: `**tekst**`
- Kursywa: `*tekst*`
- Kod inline: `` `kod` ``
- Linia pozioma: `---`
- NIEobsługiwane: linki `[]()`, obrazy, bloki kodu potrójnym backtickiem, zagnieżdżenie list głębsze niż 1 poziom

## Zasady unikalności slugów

- `course.slug` musi być unikalny wśród wszystkich zaimportowanych kursów.
- `module.slug` musi być unikalny w obrębie kursu.
- `item.slug` musi być unikalny w obrębie CAŁEGO modułu, niezależnie od
  typu — artykuł, quiz i egzamin w tym samym module NIE MOGĄ mieć tego
  samego sluga. Dobra konwencja: dla quizu użyj sluga poprzedzonego
  `quiz-` (np. artykuł `wprowadzenie`, jego quiz `quiz-wprowadzenie`).
- Używaj tylko małych liter, cyfr i myślników w slugach, np. `"wprowadzenie-do-tematu"`.
- Slug ostatniego (egzaminacyjnego) modułu powinien brzmieć `exam`.

## Co zrobić z wygenerowanym plikiem

1. Zapisz odpowiedź AI jako plik `.json` (dowolna nazwa, np. `moj-kurs.json`).
2. Przejdź na stronę **/courses/import** platformy learn.
3. Wgraj plik — kurs zostanie zapisany lokalnie w przeglądarce (local storage)
   i natychmiast dostępny pod adresem `/courses/{slug}` z pełną
   funkcjonalnością: przeglądanie modułów, artykuły, quizy jedno- i
   wielokrotnego wyboru, quiz końcowy oraz śledzenie postępu.
